# Accenture Online Voting System – Technology Consulting Simulation

This repository contains my work from the Accenture Technology Consulting Virtual Experience Program hosted on Forage.

The project simulates a consulting engagement where Accenture is tasked with designing a secure Online Voting System for the Australian Federal Government.

The objective was to analyze system requirements, design key modules, and create Agile user stories for the platform.

---

# Project Objectives

Design a secure online voting platform capable of:

• Managing voter eligibility  
• Creating and publishing ballots  
• Allowing secure vote submission  
• Recording votes securely  
• Providing audit reporting and election results

## Task 2 — Requirements Gathering

In this stage, functional requirements for the Ballot Administration module were analyzed.

Stakeholder questions were created to clarify system behavior related to:

• Ballot creation and editing  
• Access permissions for administrators  
• Voter eligibility rules  
• Ballot lifecycle management  
• Security and audit requirements  

#### 📂 [Task 2 - Files](https://github.com/VishwasJadhav/Accenture-Online-Voting-System-Consulting-Simulation/tree/main/Task%202%20-%20Requirements%20Gathering)

---

## Task 3 — System Design

A high-level design for the Ballot Administration module was developed.

The design includes:

### Functional Modules
• Ballot Authentication  
• Ballot Header Management  
• Ballot Line Item Configuration  
• Workflow Approval System  
• Ballot Publishing Controls  

### Process Flow
1. Administrator logs into the ballot administration system  
2. Ballot is created and configured  
3. Voting rules and eligibility are defined  
4. Ballot is submitted for approval  
5. Election authority reviews the ballot  
6. Approved ballots are published  
7. Voters access ballots through the voting portal  

### Data Schema

Key data tables defined:

• Ballot Header  
• Ballot Line Items  
• Workflow Approvers  
• Audit Logs  
• Voting Records

## Task 4 — User Story Development

Functional requirements were converted into Agile user stories.

User stories define system features from the perspective of system users.

Example:

As a Ballot Creator  
I want to configure ballot voting dates  
So that voters can vote within the defined election period.

Users represented in the user stories include:

• Ballot Administrators  
• Election Approvers  
• Voters  
• System Administrators
